# Use Azure IoT Hub + Functions for Connected Field Service

## Context
R-FS-003 needs scalable telemetry ingestion and event evaluation.

## Decision
IoT Hub for ingestion; Azure Functions for evaluation and Dataverse writes.

## Alternatives considered
1. **Stream Analytics / Event Grid** — viable; revisit if rules grow complex.
2. **Always-on hosted worker** — higher idle cost than serverless. Rejected for MVP.

## Consequences
- Positive: elastic, pay-per-use, least-privilege via Entra ID.
- Negative: cold starts; Dataverse throttling must be handled.
- Reversibility: moderate.

## Revisit trigger
Sustained high-throughput telemetry or complex windowed rules.
