# Equipment telemetry auto-creates predictive maintenance work orders

## Statement
Device telemetry flows through Azure IoT Hub; an Azure Function evaluates it and,
when a threshold is crossed, creates a work order in Dynamics 365 Field Service
(Connected Field Service) so a technician is dispatched before failure.

## Acceptance criteria
- [ ] Given telemetry above the alert threshold, when evaluated, then exactly one
      open work order is created for the asset (deduplicated).
- [ ] Given telemetry below threshold, when evaluated, then no work order is created.
- [ ] Threshold and dedup window are configurable. *(blocked by Q-FS-301)*

## Open questions
- [[Q-FS-301]] — what telemetry threshold and dedup window trigger a work order?
