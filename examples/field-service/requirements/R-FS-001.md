# Auto-schedule work orders to the best-fit technician via RSO

## Statement
When a work order is created in Dynamics 365 Field Service, Resource Scheduling
Optimization proposes and books the optimal technician based on required skills
(characteristics), territory, working hours, and current load.

## Acceptance criteria
- [ ] Given a new work order with skills and a location, when RSO runs, then a
      bookable resource booking is created within the SLA window.
- [ ] Given no eligible resource, when RSO runs, then the work order is flagged
      for manual dispatch (never silently unassigned).
- [ ] Scheduling honors technician working hours and territory boundaries.

## Rationale
Manual scheduling (see CS-FS-001) does not scale to the mobile workforce.
